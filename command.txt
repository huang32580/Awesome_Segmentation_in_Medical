#操作说明
如果要修改vit架构本身的信息，前往src/models/ViT_based/jepa_upernet/configs修改对应的yaml文件即可
config.json需要注意的参数
"checkpoint_dir"：因为脚本写的是会先去"checkpoint_dir"下面找有没有五折的权重，如果有则跳过训练直接test,因此要重复跑实验的话需要修改"checkpoint_dir"，前往config.json中修改
"freeze_mode": "none"是不冻结encoder/backbone,"encoder"是冻结encoder/backbone
"use_pad": false是采用直接resize到224，true是按照长宽比resize到224，jepa预训练是直接resize因此推荐false
 "force_prepare": true是会再重新运行数据预处理划分五折验证csv文件，false则是按照现有的csv文件运行，同一个环境下建议是false，首次运行可以是true.不过直接上面运行数据集预处理脚本就能直接划分csv，一直开着false就行。

"jepa_args": {
    "cfg": "./src/models/ViT_based/jepa_upernet/configs/jepa_vits16_224.yaml",
    "PRETRAIN_CKPT": "./pretrained_models/jepa_BUS-ep300.pth.tar"
  },

  "usfm_args": {
    "cfg": "./src/models/ViT_based/jepa_upernet/configs/usfm_vitb16_224.yaml",
    "PRETRAIN_CKPT": "./pretrained_models/USFM_latest.pth"
  }
}上面是两个模型的相关配置，如果不需要ckpt权重，改为null就行。

当 decoder_type 为 "SegViT" 时 必须搭配 "ATMLoss"。

当 decoder_type 为 "UPerHead" 时 必须搭配 "DiceBCELoss"（或其他标准的 CNN 损失函数）。
  "loss": {
    "type": "DiceBCELoss",
    "args": {}

  "loss": {
    "type": "ATMLoss",
    "args": {
      "num_classes": 1,
      "dec_layers": 3,
      "mask_weight": 20.0,
      "dice_weight": 1.0,
      "cls_weight": 1.0
    }
  },
